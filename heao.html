Network Tutorial

A detailed, step-by-step journey through professional networking.

1. Getting Started

A computer network is the backbone of modern communication. To manage professional hardware, we use the Cisco Internetwork Operating System (IOS).

2. Understanding the 4 Cisco Modes

Before configuring, you must recognize the prompt to know your access level:

Mode 1: User EXEC Mode (Router>)

Default entry mode. Limited to basic monitoring and viewing statistics.

Mode 2: Privileged EXEC Mode (Router#)

Accessed via the enable command. Allows full viewing of configurations and file management.

Mode 3: Global Configuration Mode (Router(config)#)

Accessed via configure terminal. Used for commands that affect the entire system.

Mode 4: Specific Configuration Sub-modes (Router(config-if)#)

Used for specific components like interfaces (ports) or virtual lines.

3. Complete Router Configuration Steps

Step 1 Mode Navigation

Move from User mode to Global Configuration mode to begin your setup.

Router> enable # Enters Privileged EXEC Mode
Router# configure terminal # Enters Global Config Mode

Step 2 Assigning Identity & Security

Identify the router on the network and protect it with an encrypted administrative password.

Router(config)# hostname Core_Router
Core_Router(config)# enable secret class123 # Encrypts the enable password

Step 3 Interface Configuration (Mode 4)

Assign an IP address to a physical port and manually activate the hardware.

Core_Router(config)# interface gigabitEthernet 0/0 # Enters Interface Sub-mode
Core_Router(config-if)# ip address 192.168.1.1 255.255.255.0
Core_Router(config-if)# no shutdown # Turns the port "ON"
Core_Router(config-if)# exit

Step 4 Saving Your Progress

Cisco devices use RAM for active changes. You must copy these to the NVRAM (Startup Config) to prevent data loss on reboot.

Core_Router(config)# end # Returns to Privileged Mode (#)
Core_Router# copy running-config startup-config # Saves permanently

4. Testing Connectivity (Ping)

Once configured, use the ping command to test if your router can reach other devices.

Core_Router# ping 192.168.1.10
Sending 5, 100-byte ICMP Echos to 192.168.1.10, timeout is 2 seconds:
!!!!!
Success rate is 100 percent (5/5)
